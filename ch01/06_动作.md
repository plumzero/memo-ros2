在 ROS2 中，动作是通信类型之一，专为长时间运行的任务而设计。它由三部分组成: 目标、反馈和结果。

动作基于主题和服务构建，功能与服务类似，但动作是可抢占的(可以在执行过程中取消)。此外，动作提供持续的反馈，而服务只返回一次响应。

动作使用客户端-服务器模型，类似于发布者-订阅者模型。"Action 客户端"节点向"Action 服务器"节点发送目标，后者确认目标并返回反馈流和结果。

目标:
- 能够通过`ros2 action list -t`命令查看当前活动所有动作名称及动作类型。
- 能够通过`ros2 action info <action_name>`命令查看某个动作(例如 /turtle1/rotate_absolute)的信息(例如动作类型、动作客户端数量和动作服务端数量)。
- 能够通过`ros2 interface show <action_type>`命令查看某个动作类型(例如 turtlesim/action/RotateAbsolute)的接口签名信息。
- 能够通过`ros2 action send_goal <action_name> <action_type> <values>`命令发送一个动作。

命令行发送动作示例。

控制海龟的旋转方向:
```s
ros2 action send_goal /turtle1/rotate_absolute turtlesim/action/RotateAbsolute "{theta: 1.57}"
```

在发送控制旋转动作的同时，也查看此动作的反馈:
```s
ros2 action send_goal /turtle1/rotate_absolute turtlesim/action/RotateAbsolute "{theta: -1.57}" --feedback
```
